## ğŸ—ï¸ Architecture

KaspaFlow consists of three main layers:

1. **Stream Engine (`lib/stream/engine.ts`)**
   - Handles the scheduling of micro-transactions based on flow rate.
   - Manages state (active, paused, completed) and persistence via `localStorage`.
   - Supports both real on-chain execution and demo simulation.

2. **Wallet Integration (`lib/kaspa/wallet.ts`)**
   - Wraps the `window.kasware` API for connection and signing.
   - Listens for account and balance changes.
   - Falls back to `Demo Mode` for judges without the extension.

3. **Visualization Layer (`components/dashboard/*`)**
   - **ParticleFlow**: Canvas-based animation where speed/density = flow rate.
   - **TxTicker**: Real-time feed of confirmed transaction hashes.
   - **StatsCounter**: Smoothly animated counters for global stats.
